## Содержательная часть

### Установка курса *(проговорить явно — см. [[Основные установки курса]], слайд [[Слайд — ВКР и ML System Design Doc]])*

- [ ] **Одна тема:** проект магистратуры (ВКР) и ML System Design Doc курса — **один и тот же проект**
- [ ] **На зачёте** — **презентация и защита ML System Design Doc с зачётом**; doc **под внедрение** — см. [[Критерии зачёта — ML System Design Doc]]
- [ ] **Разница с ВКР:** у диплома **жёсткий дедлайн** защиты; **внедрение** может занять **дольше магистратуры** (причины часто **не от студента**) — doc это учитывает
- [ ] Курс учит мыслить про **реальный продукт и внедрение**, а не только про **сдачу в срок** по правилам вуза

- [ ] Рассказать план: design doc, что будет зачётом, план лекций и семинаров
- [ ] Вводная часть: на примере каких **3-х проектов** будем работать ([[Кейс DDS — закупки]], [[Кейс финансы и риски]], [[Кейс GPT-внедрение]])
- [ ] Кривые хайпа — обзор
- [ ] Отличие MVP от полного продукта
- [ ] Рассказать про книги и ресурсы, на которых строится курс
- [ ] Проговорить, как устроена последняя часть курса; важность присутствия на семинарах
- [ ] Рассказать про важность **вайб-семинара (лек. 9)** и **двух семинаров по doc (лек. 10–11)** — готовят к **презентации и защите SSD с зачётом** ([[Критерии зачёта — ML System Design Doc]])
