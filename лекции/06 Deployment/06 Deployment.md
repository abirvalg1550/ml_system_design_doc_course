## Модуль: риски этапа

> **Этап:** Deployment · **SDD:** §4.1–4.2 · Предыдущий: [[05 Evaluation#Модуль: риски этапа|Evaluation]] · Следующий: [[07 Monitoring#Модуль: риски этапа|Monitoring →]]
>
> **Порядок на лекции:** **~10 мин интерактив** — студенты **сами** накидывают риски этапа → **затем** слайд с таблицей ниже → сверка → упражнение в doc.

| Риск | Вопрос для проверки | Смежник | Куда в doc |
|------|---------------------|---------|------------|
| **Нет пути к пользователю** | Excel / batch / UI — **кто** и **когда** получает? | PO, операции | §4.1, §1.2 TO-BE |
| **IT не деплоит** | Есть слот, API, среда? | IT, DevOps | §4.1, §4.6, roadmap |
| **SLA нереалистичен** | Latency / QPS согласованы? | IT, product | §4.3 |
| **Нет fallback** | Что при падении модели / сервиса? | IT, DS | §4.1, §4.6 |
| **«Кнопка AI»** | Интеграция в процесс, не виджет | Product, операции | §4.1 |
| **Стоимость** | GPU / лицензии / люди — кто платит? | PO, finance | §4.6, §3.3 |
| **Безопасность** | API, secrets, доступы | ИБ, IT | §4.4–4.5 |
