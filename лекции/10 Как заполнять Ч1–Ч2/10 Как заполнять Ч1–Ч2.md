### §1 Цели и предпосылки

#### §1.1 Зачем идём в разработку продукта?

- [ ] Бизнес-цель (`Product Owner` → **научный руководитель / заказчик**)
- [ ] Почему станет лучше с ML
- [ ] Критерии успеха итерации с точки зрения бизнеса

#### §1.2 Бизнес-требования и ограничения

- [ ] БТ и ссылки на детальные документы
- [ ] Бизнес-ограничения, ожидания от итерации
- [ ] Бизнес-процесс пилота (TO-BE) — **как модель используется**, кто пользователь
- [ ] Критерии успешного пилота — **под внедрение**; см. [[Критерии зачёта — ML System Design Doc]]

#### §1.3 Scope итерации

- [ ] Какие БТ закрываем / не закрываем (non-goals)
- [ ] **Качество и формат кода на планируемом этапе** — заявка в doc, **не код к зачёту**
- [ ] Технический долг

#### §1.4 Предпосылки решения

- [ ] Данные, горизонт, гранулярность — с обоснованием

### §2 Методология

#### §2.1 Постановка задачи

- [ ] Тип задачи ML с технической точки зрения

#### §2.2 Блок-схема решения

- [ ] Baseline и MVP — одна или две схемы
- [ ] Схема **конкретна для задачи**, не шаблонна — [product_schema](https://github.com/IrinaGoloshchapova/ml_system_design_doc_ru/blob/main/product_schema.png)

#### §2.3 Этапы решения задачи

- [ ] **Этап 1 — требования к данным** — цепочка **§2.2 → итоговые датасеты → сырые данные → доступ**:
  - поля target/features, **гранулярность**, ключи;
  - сырые данные — поля и гранулярность сырья;
  - **должность · подразделение · продукт** *(смежники)*; **как получить доступ**;
  - Data Selection — train/val/test; см. [[03 Data Understanding + DataSelection|лек. 3]]
- [ ] **Этапы 2+** — отдельно **baseline | MVP**:
  - метрики ↔ бизнес; риски и plan B
  - feature engineering, алгоритм, CV *(описание, не код к зачёту)*

## Мемы для лекции

- **PO vs DS** — «PO пишет зачем, DS пишет как; перепутали — шаблон».
- **Non-goals** — scope creep: «а давайте LLM в пилот прогноза полок».
- **Шаблонная блок-схема** — «одна схема на все продукты» = ошибка §2.2.
- **Baseline vs MVP** — сидят на baseline, называют MVP.
- **§2.3 без требований к данным** — «написали модель, не сказали какие поля нужны».
